## ERD, 데이터 모델링

- History 같은 경우, '나의 위치'를 불러올 때마다, 저장을 하는 것이라서 wifiInfo 테이블과는 크게 상관없다고 판단했다
  - '나의 위치'를 기준으로 wifiInfo에서 가까운 20개의 와이파이를 찾으면 된다
  - 그것을 연관 지어서 데이터를 저장하고, 그럴 필요는 없다
